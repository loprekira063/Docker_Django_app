Django
Your Web Application

Django is a Python web framework that handles your business logic, database, URLs, and HTML templates. It's the actual product — the thing users interact with. Think of it as the "engine" of your app.

// project structure
myproject/
├── manage.py         # run commands
├── requirements.txt  # python deps
├── myproject/
│   ├── settings.py   # config
│   └── wsgi.py       # server entry
└── myapp/
    ├── views.py      # logic
    └── urls.py       # routes
Python ORM for database Gunicorn = WSGI server
🐳
Docker
Container = Packaged App

Docker wraps your Django app + Python + all dependencies into a single portable image. It solves the classic "works on my machine" problem — if it runs in Docker locally, it runs identically on EC2.

// Dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["gunicorn", "myproject.wsgi:application",
     "--bind", "0.0.0.0:8000"]
docker build -t myapp . docker run -p 8000:8000 myapp
🐙
GitHub
Source of Truth

GitHub stores your code and Dockerfile. Your EC2 server will git clone or git pull from here to get the latest version of your app. It acts as the bridge between your laptop and the server.

// push code from laptop / pull on server
# on your laptop
git add .
git commit -m "add dockerfile"
git push origin main

# on EC2 server
git clone https://github.com/you/repo
# or to update later:
git pull origin main
Version control Free code hosting CI/CD ready later
☁️
AWS EC2
Your Server in the Cloud

EC2 is a virtual machine rented from Amazon. It runs 24/7 with a public IP. You SSH into it, install Docker, pull your code from GitHub, build the image, and run the container — your app is now globally accessible.

// ec2 setup commands
# SSH into your instance
ssh -i key.pem ubuntu@<your-ec2-ip>

# install docker + git
sudo apt update && sudo apt install -y docker.io git
sudo systemctl enable docker
sudo usermod -aG docker ubuntu

# clone, build, run
git clone https://github.com/you/repo && cd repo
docker build -t myapp .
docker run -d -p 8000:8000 myapp
t2.micro free tier Ubuntu 22.04 Port 22 + 8000 in SG
🌐
Live & Accessible
The World Can See It

Once Docker runs your container on EC2 with port 8000 exposed, anyone can visit http://<ec2-ip>:8000 and see your Django app. The container keeps it isolated, reproducible, and easy to update.

// update flow after code changes
# laptop → push
git push origin main

# EC2 → pull + rebuild
git pull origin main
docker stop myapp-container && docker rm myapp-container
docker build -t myapp .
docker run -d -p 8000:8000 --name myapp-container myapp

# visit http://<ec2-ip>:8000 ✅

// step-by-step breakdown — click to expand
▼
01
Write Your Django App
Create your project with views, models, URLs. Add requirements.txt with django and gunicorn.
django-admin startproject myproject
cd myproject
python manage.py startapp myapp
# add gunicorn to requirements.txt
▼
02
Write the Dockerfile
Tell Docker how to package your app — base image, install deps, copy code, expose port, run gunicorn.
FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["gunicorn", "myproject.wsgi:application", "--bind", "0.0.0.0:8000"]
▼
03
Push to GitHub
Initialize a git repo and push everything including the Dockerfile. EC2 will pull from here.
git init && git add .
git commit -m "initial"
git remote add origin <url>
git push -u origin main
▼
04
Launch EC2 Instance
Launch Ubuntu t2.micro on AWS. Open ports 22 (SSH) and 8000 in Security Groups. Download your .pem key.
# AWS Console → EC2 → Launch
OS: Ubuntu 22.04 LTS
Type: t2.micro (free tier)
SG: open port 22, 8000
▼
05
Install Docker on EC2
SSH into the server and install Docker + Git. Enable Docker to start on boot automatically.
sudo apt update
sudo apt install -y docker.io git
sudo systemctl enable docker
sudo usermod -aG docker ubuntu
▼
06
Build & Run the Container
Clone your repo on EC2, build the Docker image from the Dockerfile, then run it. App is live!
git clone <your-github-url>
cd myproject
docker build -t myapp .
docker run -d -p 8000:8000 myapp
# visit http://<ec2-ip>:8000 ✅
